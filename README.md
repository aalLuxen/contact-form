## Overview

### The challenge

Users should be able to:

- Complete the form and see a success toast message upon successful submission
- Receive form validation messages if a required field has been missed or the email is not formatted correctly
- Complete the form only using their keyboard
- View the optimal layout depending on their device's screen size
- See hover and focus states for all interactive elements

### What I learned

I learned how to properly structure HTML forms using semantic elements like `<fieldset>` and `<legend>` for accessibility. I also got more comfortable working with CSS Grid for layout, and learned how to create CSS animations with `@keyframes`. On the JavaScript side, I practiced DOM manipulation, event listeners, and working with classes to show and hide elements.

### Continued development

I want to keep improving my JavaScript skills. I relied quite a bit on guidance during this project, so I'd like to get more confident writing JS logic independently in future projects.

### AI Collaboration

I used Claude AI as an assistant throughout this project. The challenge came with a custom prompt that instructed Claude to guide me rather than give me direct answers, which helped me actually learn instead of just copying code. Claude helped me when I didn't remember a semantic HTML tag, explained CSS concepts like specificity and `grid-column`, and guided me through JavaScript validation logic step by step.
